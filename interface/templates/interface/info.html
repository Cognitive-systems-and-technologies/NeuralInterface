Общая информация

Позволяет работать с данными из базы данных. Полностью управлять данными можно только для таблиц с агентами и группами агентов. Позволяется создавать, изменять и удалять записи. При попытке удалить запись, которая связана с другой записью в другой таблицы посредством внешнего ключа, система выдаст предупреждение о том, что сделать это нельзя и необходимо сначала удалить все данные таким образом, чтобы у БД сохранялась структура третьей нормальной формы. Осуществляется всё посредством кнопок. При создании и изменении открывается модальное окно для заполнения или изменения данных соответственно. Таблица типов агентов статичная и содержит значения по умолчанию. Остальные таблицы, если есть необходимость, нужно заполнять через административную панель сайта, которая позволяет полностью изменять данные.

Работа с агентами
Важно! Для корректной работы веб-приложения, необходимо, чтобы MAC адреса и IP адреса у агентов были уникальные. Данные в таблицах связаны внешними ключами. Вы не сможете удалить связанные данные. Например, Вы не сможете удалить агента, если есть данные в таблице с файлами для этого агента или в какой-нибудь другой таблице.
Работа с группами
Важно! На группы агентов действуют схожие ограничения, что и на самих агентов. Например, Вы не сможете удалить группу агентов, если к ней принадлежит хотя бы один агент.
Общая информация

Выводит информацию об обучении агентов, а также позволяет управлять данными агентов. По умолчанию, всегда выбирается первая запись из таблицы ошибок агентов. График отображает данные обучения агентов. При выборе необходимого агента в таблице, график перерисовывается, исходя из значений ошибок для определенного агента. Панель в правой нижней части экрана позволяет отправлять необходимые команды агентам.

Работа с графиком

Кнопки в правой нижней части страницы отвечают за прямое управление и взаимодействие с агентами. При нажатии на кнопку отправляется запрос, который в зависимости от состояния агента отправляет определенный ответ. Как показано на видео, сервер не может найти агента в локальной сети с таким адресом и присылает ошибку, которая в полном виде отображается в нижней части страницы.

Важно! График не обновляется автоматически. Для обновления данных нажмите на кнопку обновления графика в верхней части экрана или на строчку с агентом в таблице

Общая информация

Административная панель Django - это встроенный инструмент, предоставляемый Django для управления данными вашего веб-приложения. Она обеспечивает удобный и мощный интерфейс для работы с моделями и данными в вашей базе данных без необходимости создания собственного пользовательского интерфейса. Это мощный инструмент, который значительно упрощает управление данными в вашем веб-приложении, и она поставляется с Django "из коробки", что делает ее одним из популярных выборов для административного интерфейса.

Вход в административную панель
  1. Зайдите на страницу административной панели
    Откройте веб-браузер и введите адрес административной панели (обычно это /admin) в адресной строке сайта или нажмите в левом меню "Панель администратора"
  2. Авторизуйтесь
    Вам будет предложено ввести учетные данные. Введите логин и пароль, указанные выше
Возможности панели администратора

Аутентификация и авторизация: Административная панель предоставляет механизмы аутентификации для входа в систему. Пользователь должен ввести правильные учетные данные (логин и пароль), чтобы получить доступ к административной панели. Кроме того, она поддерживает систему авторизации, что позволяет управлять правами доступа пользователей, включая суперпользователя.

Доступ к моделям приложения: Административная панель автоматически анализирует ваше Django-приложение и создает интерфейс для работы с моделями, определенными в приложении. Она позволяет вам просматривать все модели, доступные в вашем приложении, и взаимодействовать с ними.

CRUD операции: CRUD - это аббревиатура, означающая "Create", "Read", "Update" и "Delete". Административная панель позволяет выполнять все эти операции с данными вашей базы данных. Вы можете создавать новые записи, просматривать их, обновлять значения полей и удалять записи.

Фильтрация, поиск и сортировка: Административная панель предоставляет удобные средства для фильтрации данных, поиска и сортировки записей. Это очень полезно, когда у вас есть большое количество данных и вы хотите быстро найти нужные записи.

Валидация данных: При создании или редактировании записей, административная панель выполняет проверку валидности данных, чтобы убедиться, что введенные значения соответствуют определенным правилам и ограничениям.

Отображение связей: Если у ваших моделей есть взаимосвязи через внешние ключи или другие отношения, административная панель отображает эти связи и позволяет легко перемещаться между связанными записями.

Настройка интерфейса: Вы можете настроить внешний вид и функциональность административной панели, добавлять или скрывать поля, настраивать формы и управлять внешним видом моделей.

Важно! С осторожностью удаляйте, изменяйте или добавляйте данные! Нарушив целостность структуры базы данных может полечь за собой нестабильную работу всего приложения. Если же Вам всё же нужно ка-то изменить структуру базы данных, добавить новые таблицы или изменить внешние ключи, то рекомендуем пользоваться бесплатной программой Dbeaver.

Наименование проекта

Библиотека глубокого обучения для мобильных роботов и роботизированных систем на ARM-процессорах

Цель выполнения проекта

Разработка библиотеки для глубокого иерархического обучения интеллектуальных агентов (мобильных роботов и роботизированных систем) на базе ARM процессоров.

Задачи по проекту
  1. Создание ядра библиотеки для глубокого обучения. Создание основных нейросетевых структур (разные типы слоев, сетей и алгоритмов их обучения).
  2. Реализация алгоритмов самообучения интеллектуальных агентов. Реализация алгоритмов глубокого обучения с подкреплением.
  3. Реализация алгоритмов мультиагентного взаимодействия. Реализация алгоритмов взаимодействия агентов внутри мультиагентных систем. Реализация алгоритмов обмена сообщениями между интеллектуальными агентами при работе в группах.
  4. Создание модулей визуализации и удаленного мониторинга. Создание пользовательских локальных и веб интерфейсов для управления и мониторинга процессами работы нейросетевых алгоритмов на аппаратной части.
Область применения научно-технического продукта и т.п.
  • Системы беспилотных автоматических наземных аппаратов применяемых для доставки небольших грузов (интеллектуальные роботы-доставщики).
  • Роботы-конструкторы для обучающих комплексов.
  • «Умные» автономные и работающие совместно приборы для систем умного дома (регулируемые датчики, выключатели, регулировщики) и интернета вещей.
  • Сфера развлечений: групповые игры в соревнованиях по робототехнике.
  • Наука: создание прототипов многоагентных роботизированных систем для изучения и разработки методов коллективного взаимодействия.
